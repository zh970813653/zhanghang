    # TypeScript

    TypeScript是JavaScript的一个超集，主要提供了类型系统和对ES6的支持，它与2012年10月正式发布第一个版本。

    优势：

    1. 能在开发过程中更快的发现潜在问题。
    2. 对编辑器更友好的代码提示功能。
    3. 代码语义更清晰易懂。

    # 安装

    # 安装命令
npm install -g typescript

# 安装完毕后，查看版本号

    类的访问修饰符

  protected gender: boolean

    this.age = age
    this.gender = true
  }
  sayHi (msg: string) : void {
    console.log(`I am ${this.name},${msg}`)
    console.log(this.age) // 这里可以访问
  }
}
const tom = new Person ('zhanghang',18)

    # 泛型

    泛型generics是指在定义函数、接口和类的时候，不预先指定其具体类型，而在使用的时候再去指定的一种特性 函数中的泛型 假设我们有如下一个函数，其中参数a和b接受的类型必须为相同的类型。

    function join (a, b) {
  return `${a}${b}`
}
    1
    2
    3

    我们在没有了解到泛型之前，我们可以用联合类型来定义：

    function join (a: number | string, b: number | string) {
  return `${a}${b}`
}
    1
    2
    3

    在以上的例子中，我们仅仅只是规定了a和b参数必须是number类型或者string类型，但并没有办法来限制a和b必须是同一个类型。这个时候我们可以使用泛型来表示：

    function join<T> (a: T, b: T): string {
  return `${a}${b}`
}
console.log(join(1, 2))     // 12    
console.log(join('1', '2')) // 12
console.log(join(1, '2'))   // 编译报错
    1
    2

    注意：我们在调用join()函数并进行传参的时候，TypeScript会自动帮我们推断参数的类型，以上三行代码也可以像如下方式进行撰写

    console.log(join<number>(1, 2))     // 12    
console.log(join<string>('1', '2')) // 12
console.log(join<number>(1, '2'))   // 编译报错

    注意

    泛型可以是多个的。

    function join<T, P> (a: T, b: P): string {
  return `${a}${b}`
}
console.log(join(1, 2))     // 12    
console.log(join('1', '2')) // 12
console.log(join(1, '2'))   // 12
