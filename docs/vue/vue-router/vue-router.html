    # Vue Router 实现原理

    # Hash模式

    1. Url中#后面的内容作为路径的地址
    2. 监听hashchange事件
    3. 根据当前路由地址找到对应组件重新渲染

    # History 模式

    1. 通过history.pushState() 方法改变地址栏
    2. 监听popstate事件
    3. 根据当前路由地址找到对应组件进行渲染

    # VueRouter 模拟实现-分析

    首先我们先回顾一下核心代码

    代码分析

    1. vue.use这个方法它可以传入一个对象或者函数。如果传入函数的话，vue.use内部会直接调用这个函数。如果传入的是一个对象的话，vue.use内部会调用这个对象内部的install方法。 所以接下来我们还需要自己手动去实现一个install方法。
    2. 当我们new VueRouter的时候，也就是创建一个vue实例的时候可知VueRouter会是一个构造函数或者是一个class类，因此一会我们用class类的方式去实现，并且这个类应该有一个静态的install方法让vue.use内部进行一个调用。VueRouter的构造函数需要接受一个参数，并且这个参数是一个对象，里面可以配置路由的一些规则。
    3. 最后创建一个vue的实例。并且在vue的实例里面传入我们已经创建好的router对象

    类图解析

    类图前三个为属性，后面的为方法。 +代表对外公开的方法。 -代表静态的方法

    1. options这个属性的作用是记录构造函数中传入的对象(new VueRouter时传入的对象)
    2. data是一个对象。它里面有一个属性current。current这个属性是用来记录当前的路由地址。此处设置一个data属性的目的是我们需要一个响应式的对象。也就是data是响应式的对象。因为当路由的地址发生变化后，地址对应的组件需要发生自动的更新。。如何把对象变成响应式的? 需要调用Vue.observable()这个方法
    3. routerMap是一个对象，用来记录我们路由地址和组件的对应关系
    4. install就是一个静态的方法，它是用来实现vue的插件机制
    5. 构造函数Constructor帮我们初始化刚刚提到过的三个属性。
    6. init方法用来调用initEvent方法、createRouterMap方法、initComponents方法。这里把不同的代码分割到不同的方法中去实现
    7. initEvent方法用来注册popstate事件。用来监听浏览器历史的变化
    8. createRouterMap方法用来初始化routerMap这个属性的。它把构造函数constructor传入的路由规则转换成键值对的形式存到routerMap中去。
    9. initComponents方法它是用来创建roouter-link和router-view这两个组件的

    # VueRouter-install

    利用vue-cli初始化一个vue项目。然后自己实现一个VueRouter

    // |-- src
// |-- |-- myRouter
// |-- |-- |-- index.js

let _Vue = null
class VueRouter {
  static install (Vue) {
    /**
     * install需要做的事情
     */
    // 1. 判断当前插件是否已经安装，如果已经安装，则不需要再次安装了
    // 函数也是对象，对象就有属性，给当前install记录一个属性表示插件已经安装
    if (VueRouter.install.installed) return
    VueRouter.install.installed = true
    // 2. 把Vue的构造函数记录到全局变量中去
    _Vue = Vue
    // 3. 把创建Vue实例时候传入router对象注入到vue实例上
    // mixin 混入
    _Vue.mixin({
      beforeCreate () {
        // 只有vue实例才调用 组件是不需要调用的
        // 判断$options.$router 因为只有在vue的实例才会有router
        if (this.$options.router) {
          _Vue.prototype.$router = this.$options.router
        }
      }
    })
  }
}

export default VueRouter

    # VueRouter-构造函数 constructor

    let _Vue = null
class VueRouter {
  constructor (options) {
    this.options = options
    /**
     * routerMap是一个对象，这个对象的作用就是将来把我们传入的options也就是路由规则给解析出来存储到routerMap中去
     * routerMap的键存的是路由地址，值就路由组件。 将来在router-view这个组件里面会根据当前的路由地址在routerMap中找到对应的组件进行渲染
     */
    this.routerMap = {}
    this.data = _Vue.observable({
      current: '/' // current的作用是用来存储当前的路由地址
    })
  }
}

export default VueRouter

    # VueRouter-createRouterMap

    这个方法的作用就是将我们传入的options中的routes路由规则解析成键值对的形式，存储到routerMap中去

    let _Vue = null
class VueRouter {
  constructor (options) {...
  }
  static install (Vue) {...
  }
  createRouterMap () {
    // 这个方法的作用就是将我们传入的options中的routes路由规则解析成键值对的形式，存储到routerMap中去
    this.options.routes.forEach(route => {
      this.routerMap[route.path] = route.components
    })
  }
}
export default VueRouter

    # 方法2: 用render函数解决

    从以上遇到的问题我们知道运行时版本是不支持template的。那我们为什么在平常开发中的模板都是写在template里面并且不报错的呢?
    因为vue在打包的时候会帮我们将template模板编译成render函数，这个过程也叫预编译
    首先对目录的vue.config.js进行删除，并重启项目

    // 对initComponents函数进行修改
 initComponents (Vue) {
   Vue.component('router-link', {
     props: {
       to: String
     },
     // h函数的作用是帮我们创建虚拟dom render函数的作用就是调用这个h函数并把结果进行返回
     render (h) {
       return h('a', {
         // dom 对象属性
         attrs: {
           href: this.to // 不需要拼# 因为我们实现的是history模式
         }
       }, [this.$slots.default])
     }
     // template: '<a :href="to"><slot></slot></a>'
   })
 }

    # VueRouter-router-view

    在以上代码的基础上，我们进行运行项目进行a链接点击测试发现:浏览器进行了一个刷新。这是因为a标签点击的时候会默认想服务器发送请求
    接下来我们对router-link组件进行一个修改，并对router-view组件进行一个创建

    initComponents (Vue) {
    Vue.component('router-link', {
      props: {
        to: String
      },
      // h函数的作用是帮我们创建虚拟dom render函数的作用就是调用这个h函数并把结果进行返回
      render (h) {
        return h('a', {
          // dom 对象属性
          attrs: {
            href: this.to // 不需要拼# 因为我们实现的是history模式
          },
          on: {
            click: this.clickHandle
          }
        }, [this.$slots.default])
      },
      methods: {
        clickHandle (e) {
          // 当我们点击a链接的时候要做两件事
          // 1. 调用history.pushState方法,并将需要跳转或者跳转后的连接给到路由对象的data属性上的current属性上
          history.pushState({}, '', this.to)
          this.$router.data.current = this.to
          // 2. 阻止a链接的默认行为，使其不再像服务器发送请求
          e.preventDefault()
        }
      }
      // template: '<a :href="to"><slot></slot></a>'
    })
    Vue.component('router-view', {
      render: (h) => {
        const currentComponents = this.routerMap[this.data.current]
        return h(currentComponents)
      }
    })
  }

    # VueRouter-initEvent

    initEvent的作用就是监听浏览器的的路由路径前进以及后退，拿到对应的path给到current

    // 在init函数中进行初始化
  init () {
    this.createRouterMap()
    this.initComponents(_Vue)
    this.initEvent()
  }
  initEvent () {
    // popState 当历史发生变化的时候 会触发
    window.addEventListener('popstate', () => {
      // 将路径赋值给current
      this.data.current = window.location.pathname
    })
  }
